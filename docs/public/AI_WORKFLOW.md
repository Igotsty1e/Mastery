# AI Workflow

## Where AI is used in the product

- borderline `sentence_correction` evaluation when deterministic rules cannot decide cleanly
- short post-lesson debrief generation

## Where AI is used in development

- implementation support
- documentation drafting
- controlled content-authoring assistance

## Where AI fails

- it can overgeneralize on grammar-sensitive edge cases
- it can produce plausible but ungrounded explanations
- it is not reliable enough to be the primary evaluator for the core lesson flow

## Where humans intervene

- pedagogy and curriculum decisions
- lesson authoring and content review
- final decisions on product behavior and architecture
- repository readiness and documentation cleanup before any publication

## Guiding rule

AI is used as a constrained assistant. Product correctness, pedagogy, and release decisions stay with humans.
