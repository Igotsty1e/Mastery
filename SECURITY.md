# Security Policy

This repository is being prepared for possible public release, but it should be treated as an actively developed project.

## Reporting a vulnerability

Do not open a public issue for suspected vulnerabilities.

Report security issues privately to the maintainer with:

- a short summary
- affected component or file
- reproduction steps
- impact assessment
- any suggested mitigation

## Repository posture

- secrets and local environment files must never be committed
- operational deployment details should stay outside the public repository
- internal evaluation assets should be reviewed before publication

## Scope notes

- production endpoints and dashboard-specific runbooks are intentionally not documented here
- this repository may contain development-only code paths guarded from production use
